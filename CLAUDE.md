# Calendar Sidebar

Always-on-top Windows desktop sidebar showing today's Google Calendar events. PyQt6 hosts a WebEngine that renders a Svelte/SvelteKit frontend; a FastAPI/uvicorn backend runs on a background thread and handles Google OAuth2 + event CRUD.

## Architecture

```
widget.py    PyQt6 window + server thread launcher
server.py    FastAPI backend: OAuth2 flow, /api/events CRUD, SPA static serving
src/         Svelte/SvelteKit frontend (adapter-static)
build/       Compiled frontend — generated by npm run build (gitignored)
run.vbs      Silent Windows launcher (hides the console)
widget.log   Runtime log file (gitignored) — the only output when run via VBS
.env         Google OAuth credentials (gitignored)
.tokens.json Persisted OAuth tokens (gitignored)
```

## Running

```
npm run build       # compile Svelte frontend (required once)
uv run widget.py    # start the app (uv handles all Python deps)
```

Or double-click `run.vbs` for a silent, no-console launch. All log output goes to `widget.log`.

## Python deps

Declared inline in the `# /// script` block at the top of each `.py` file — `uv` installs them automatically. No `requirements.txt` or `pyproject.toml`.

## Google OAuth setup

1. Create an OAuth 2.0 credential in Google Cloud Console (type: web application).
2. Add `http://localhost:8764/api/auth/callback` as an authorised redirect URI.
3. Copy `.env.example` to `.env` and fill in `GOOGLE_CLIENT_ID` and `GOOGLE_CLIENT_SECRET`.
4. Open the app and navigate to `/api/auth/google` to complete the first-time auth flow.

## Debugging

Check `widget.log` first. Key log lines to look for:

| Log message | What it means |
|---|---|
| `Missing Google OAuth config` | `.env` is missing or incomplete |
| `Token file not found` | Not yet authenticated — visit `/api/auth/google` |
| `API request rejected — user not authenticated` | 401 on event fetch; same fix |
| `Credential refresh failed` | Refresh token revoked; re-authenticate |
| `WebEngine failed to load` | Server didn't start in 900 ms — check for port conflict |

## Conventions

- Python: `uv` only (no pip), f-strings, full type hints, loguru for all logging.
- No `import *`.
- Keep `widget.py` focused on the Qt layer; keep business logic in `server.py`.
